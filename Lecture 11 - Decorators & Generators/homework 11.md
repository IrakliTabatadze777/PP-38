თუ რომელიმე ატრიბუტი არის უარყოფითი, დააბრუნეთ ტექსტი, რომ მხოლოდ დადებით რიცხვებზე ხდება მოქმედება
სხვა შემთხვევაში გამოიძახეთ ფუნქცია და დააბრუნეთ შედეგი

### დავალება 2

დაწერეთ ორი ფუნქცია, სხვადასხვა მათემატიკური მოქმედებისთვის,
ააწყვეთ დეკორატორი, რომელსაც ორივე ფუნქციისთვის გამოიყენებთ
დეკორატორმა უნდა დააფორმატოს ფუნქციის დაბრუნებული მნიშვნელობა და დააბრუნოს

დაბეჭდვის ფორმატი: `function called '<func_name>', with attributes <num1> და <num2>, returned <returned_value>`

მაგალითი: გამოძახება `add(10, 15)` დააბეჭდავს: `called function 'add', with attributes 10 and 15, returned 25`

### დავალება 3

დაწერეთ დეკორატორი, რომელსაც ატრიბუტად გადაეწოდება ორი მნიშვნელობა: `times` და `delay`

- `times` - აღწერს თუ რამდენჯერ უნდა იყოს ფუნქცია გამოძახებული
- `delay` - აღწერს თუ რამხელა დაყოვნება უნდა იყოს ფუნქციების გამოძახებას შორის

ფუნქცია, რომლისთვისაც დეკორატორს გამოიყენებთ, შესაძლებელია ასრულებდეს მხოლოდ print ფუნქციონალს

დაყოვნებისათვის შეგიძლიათ გამოიყენოთ:
```python
import time
time.sleep(10)  # ატრიბუტად იღებს წამს
```



### დავალება 4

შექმენი გლობალური ცვლადი `current_user`, რომელიც იქნება შემდეგი სახის დიქტი:

```python
current_user = {
    "username": "irakli",
    "role": "admin"  # შეცვალე შემდეგში სხვადასხვა როლზე ტესტირებისთვის
}
```

შექმენი შემდეგი სამი ფუნქცია:

- `delete_user(user_id)` — ბეჭდავს: `User with id {user_id} has been deleted.`
- `edit_user(user_id)` — ბეჭდავს: `User with id {user_id} has been updated.`
- `create_user(first_name)` — ბეჭდავს: `User {first_name} has been created.`

შექმენი დეკორატორი სახელად `@role_required(role)`, რომელიც:

- იღებს ერთ არგუმენტს — საჭირო როლს (მაგ: 'admin', 'editor', 'user')
- ამოწმებს, აქვს თუ არა `current_user`-ს ეს როლი
- თუ როლი არ ემთხვევა → უნდა დაბეჭდოს: `Permission denied!` და არ უნდა შეასრულოს ფუნქცია
- თუ როლი ემთხვევა → ფუნქცია უნდა შესრულდეს ჩვეულებრივ
