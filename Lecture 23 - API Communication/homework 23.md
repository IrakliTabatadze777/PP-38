რეცეპტების მართვის სისტემა CrudCrud API-ით

---

## მოთხოვნები

დავალების შესასრულებლად დაგჭირდება:

- ბიბლიოთეკები: `requests`, `pydantic`
- API: [https://crudcrud.com](https://crudcrud.com)

საჭირო ბიბლიოთეკების ინსტალაცია:

```bash
pip install requests pydantic
```

---

## JSON ფაილები

### `recipes.json` — საწყისი მონაცემები (წინასწარ მოცემული)

```json
[
  { "name": "Khachapuri",  "cuisine": "Georgian", "time_minutes": "30" },
  { "name": "Khinkali",    "cuisine": "Georgian", "time_minutes": "60" },
  { "name": "Mtsvadi",     "cuisine": "Georgian", "time_minutes": "45" }
]
```

### `updated_recipes.json` — განახლებული მონაცემები (შენ უნდა შექმნა)


---

## დავალების აღწერა

### ნაწილი 1 — Pydantic მოდელი

შექმენი `Recipe` კლასი Pydantic-ის გამოყენებით:

- `name` — სტრინგი
- `cuisine` — სტრინგი
- `time_minutes` — სტრინგი

---

### ნაწილი 2 — POST: რეცეპტების დამატება

წაიკითხე `recipes.json` ფაილი და **ყველა რეცეპტი დაამატე** API-ში `POST` მეთოდით.


---

### ნაწილი 3 — GET: ყველა რეცეპტის წამოღება

გამოიტანე **ყველა რეცეპტი** API-დან და დაბეჭდე თითოეულის სახელი და მომზადების დრო.

---

### ნაწილი 4 — GET: ერთი რეცეპტი ID-ით

პირველი რეცეპტის `_id`-ის გამოყენებით **წამოიღე კონკრეტული რეცეპტი** და გაბეჭდე მისი ყველა მონაცემი.


---

### ნაწილი 5 — PUT: სრული განახლება

განაახლე ერთ-ერთი რეცეპტის ჩანაწერი და შეინახე ფაილში
> ყურადღება: `PUT` ანაცვლებს ობიექტს მთლიანად. `_id` ველი **არ უნდა გადაეგზავნოს** body-ში.


---

### ნაწილი 6 — DELETE: რეცეპტის წაშლა

წაშალე **ბოლო რეცეპტი** სიიდან.

შემდეგ კვლავ გამოიძახე `GET /recipes` და დაბეჭდე დარჩენილი რეცეპტები, რათა დაადასტურო წაშლა.

---

### ნაწილი 7 — შეცდომების დამუშავება

**ყველა** მოთხოვნა უნდა იყოს `try/except` ბლოკში და დაიჭიროს:

| Exception | რა შემთხვევაში |
|---|---|
| `HTTPError` | სტატუსი 4xx / 5xx |
| `ConnectTimeout` | სერვერი დიდხანს არ პასუხობს |
| `ValidationError` | Pydantic-ის ვალიდაცია ვერ მოხდა |
